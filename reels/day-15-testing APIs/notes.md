# Day 15 - Testing APIs

## Problem

APIs without testing lead to:

* bugs in production
* fragile systems
* fear of changes
* difficult maintenance

## Bad Practice

No tests at all.

Code is deployed without validation.

## Good Practice

Test different layers:

* Unit tests
* Integration tests
* API tests

## Testing Types

Unit Test:
Test small pieces of logic.

Integration Test:
Test interaction between components.

API Test:
Test endpoints.

## Tools

JUnit
Mockito
Spring Boot Test
RestAssured
Postman / Newman

## Benefits

* fewer bugs
* safer deployments
* better code quality
* confidence in changes

## Key Idea

Code without tests is a risk.
