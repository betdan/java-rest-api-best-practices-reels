Día 14 – Estrategia de caché en APIs.

Un error común es consultar la base de datos en cada petición.

Esto genera respuestas lentas y alta carga en el sistema.

Un mejor enfoque es usar caché.

La API primero revisa la caché y solo consulta la base de datos si es necesario.

Cliente a API, API a Caché, Caché a Base de datos.

Esto mejora el rendimiento y reduce la carga del sistema.

El caché es esencial para APIs escalables y de alto rendimiento.
